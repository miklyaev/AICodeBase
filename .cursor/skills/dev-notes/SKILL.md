# Dev Notes

## Когда применять

Применяй этот skill **в конце сессии**, когда были внесены значимые изменения в код проекта.

Если нет файла `dev_notes.md`, создать его.

Вносить в файл `dev_notes.md` ключевые изменения в проекте. Особенно это относится к изменениям в архитектуре, к изменениям на Backend, к изменениям компонентов на фронтенде. Исключения составляют небольшие изменения разметки.

## Что записывать

| Категория | Примеры | Записывать? |
|-----------|---------|-------------|
| Архитектура | новые модули, слои, паттерны, смена стека, структура папок | Да |
| Backend | API, сервисы, контроллеры, БД, RAG-пайплайн, WebSocket | Да |
| Frontend | новые/переработанные React-компоненты, store, роутинг | Да |
| Инфра | Docker, CI, env, конфиг сборки | Да |
| Разметка | правки отступов, цветов, мелкий CSS без логики | Нет |
| Мелкие правки | typo, rename переменной без смены поведения | Нет |

## Расположение файла

`dev_notes.md` — в **корне репозитория**.

## Workflow

1. Проверить наличие `dev_notes.md`. Если файла нет — создать по шаблону ниже.
2. Оценить diff текущей сессии по таблице выше.
3. Если есть что записать — добавить **новую запись в начало** секции «Журнал изменений» (сверху — самое новое).
4. Одна запись = одна логическая единица работы (фича, рефакторинг, архитектурное решение).
5. Не дублировать уже описанные изменения.

## Шаблон нового файла

```markdown
# Dev Notes

Краткий журнал ключевых изменений проекта AICodeBase.

## Журнал изменений

<!-- Новые записи добавлять сверху -->
```

## Шаблон записи

```markdown
### YYYY-MM-DD — Краткий заголовок

**Область:** Architecture | Backend | Frontend | Infra

**Что изменилось:**
- ...

**Затронутые пути:**
- `path/to/file`

**Зачем / контекст:** (1–2 предложения, если неочевидно)
```

## Примеры

### Записать

- Добавлен NestJS-модуль `IndexingModule` с LanceDB-адаптером
- Новый компонент `ChatPanel` и store на Zustand
- REST `POST /api/projects/:id/index` + SSE для прогресса

### Не записывать

- Подправлен `padding` у кнопки в `Sidebar.module.scss`
- Исправлена опечатка в комментарии
- Сдвинут блок на 4px без изменения структуры компонента

## Язык

Записи — на **русском**, кратко и по делу.
